# Monetization

Every monetization system is a module toggle. DMs (Dungeon Masters) choose which systems exist in their world. The platform enforces transparency and player protection regardless of what the DM enables.

---

## 1. RWT Marketplace (Built-In, Regulated)

Real-world trading is not banned -- it is regulated. If players are going to trade anyway, the platform provides escrow, fraud protection, and dispute resolution instead of pushing it to sketchy third-party sites.

### DM Controls

| Setting | Description |
|---|---|
| RWT Allowed | Master toggle (on/off) |
| Tradeable Categories | GP, items, accounts, services, cosmetics |
| Price Floor/Ceiling | Min and max price per category |
| Platform Cut (%) | Percentage taken by OpenScape |
| DM Cut (%) | Percentage taken by the world operator |
| Seller Verification | Identity verification required before listing |
| Buyer Protection | Escrow holds funds until delivery confirmed |
| Dispute Resolution | Platform-mediated resolution process |
| Seller Ratings/Reviews | Public feedback on completed transactions |
| Transaction History | Full audit trail for all parties |

---

## 2. Service Marketplace

Players sell their time and skill to other players. Account sharing is handled through a permission system, not trust.

### Service Types

| Service | Description |
|---|---|
| Account Training | Level skills on behalf of the owner |
| Boss Carries | Experienced player carries through boss fights |
| Quest Completion | Complete quests on the owner's account |
| Cape Services | Achieve cape milestones (fire cape, infernal, etc.) |
| Coaching | Live guidance without account access |
| Building | Player-owned house or clan hall construction |
| Gathering | Resource collection services |
| Custom | Freeform service with agreed-upon scope |

### RBAC Account Sharing

When a service requires account access, the owner grants temporary, scoped permissions. The service provider never has full account control.

| Permission | Description |
|---|---|
| View Only | Can observe but not interact |
| Skill Only | Can train specified skills, nothing else |
| Combat Only | Can fight monsters, no skilling or trading |
| Full Play | All gameplay actions permitted |
| Custom | Owner defines exact permission set |

### Account Sharing Protections

| Protection | Description |
|---|---|
| Time-Limited | Access auto-revokes after set duration |
| Activity Log | Every action recorded and visible to owner |
| Restricted Areas | Cannot access bank, drop items, or trade (configurable) |
| Gear Lock | Equipped gear cannot be removed or dropped |
| GP Lock | Gold cannot be transferred out |
| Insurance | Platform-backed coverage for rule violations |
| Kill Switch | Owner revokes access instantly from any device |
| 2FA on Sensitive Actions | Two-factor required for bank, trade, drop above threshold |

## 8. P2W Meter (Platform-Enforced Transparency)

Every world receives an auto-calculated pay-to-win label based on what the DM has enabled. DMs cannot hide or override this label. Players use it to filter the server list.

| Label | Criteria |
|---|---|
| Cosmetic Only | Nothing purchasable affects gameplay |
| Convenience | QoL purchases only (bank space, presets) |
| Accelerator | XP boosts or time-savers available |
| Advantage | Better gear, drop rates, or stats purchasable |
| Full P2W | Anything goes, no restrictions |

The label is calculated automatically from the DM's enabled modules and shop contents. If a DM sells an item that grants combat stats, the label escalates whether they like it or not.
